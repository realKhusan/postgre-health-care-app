Welcome to the Health Care Management System

This application helps you manage patients and their appointments through a RESTful API with a PostgreSQL backend.

📋 Patient Module

  • GET → /api/patients/all – Retrieve all patients
  • POST → /api/patients/add – Add a new patient
  • PUT → /api/patients/edit/:id – Edit a patient by ID
  • DELETE → /api/patients/delete/:id – Delete a patient by ID
  • GET → /api/patients/without-appointments – Get patients without appointments

📄 Patients Table Fields:

  • id – Unique identifier (BIGSERIAL, PK)
  • name – Full name (VARCHAR, required)
  • age – Age (must be ≥ 0)
  • medical_history – Medical background (TEXT)
  • phone – Contact number (VARCHAR)
  • email – Unique email (VARCHAR, unique)
  • address – Home address (TEXT)
  • created_at – Timestamp when added
  • updated_at – Timestamp when last updated

📅 Appointment Module

  • GET → /api/appointments/all – Retrieve all appointments
  • POST → /api/appointments/add – Add a new appointment
  • PUT → /api/appointments/edit/:id – Edit an appointment by ID
  • DELETE → /api/appointments/delete/:id – Delete an appointment by ID
  • GET → /api/appointments/by-date – Filter by date range
  • GET → /api/appointments/by-doctor/:doctor – View appointments by doctor

📄 Appointments Table Fields:

  • id – Unique identifier (BIGSERIAL, PK)
  • patient_id – Reference to patient (FK)
  • doctor_name – Name of the doctor (VARCHAR, required)
  • appointment_date – Date of appointment (DATE, required)
  • status – Appointment status: scheduled, completed, or canceled
  • created_at – Timestamp when added
  • updated_at – Timestamp when last updated

🛡️ Data Validation

All inputs are validated on the server to ensure:

  • ✔ Required fields (e.g. name, date) are not empty
  • ✔ Valid formats for email, date, status
  • ✔ Logical constraints (e.g. age ≥ 0, status in allowed values)

Use Postman or any REST client to interact with these endpoints.
